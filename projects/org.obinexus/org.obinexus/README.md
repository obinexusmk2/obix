## Gating: Computational cognition for real teams

Gating is how I turn intent into accountable progress across any methodology (agile, kanban, waterfall, or mixed).

- **Core gates:**  
  - **Open gate:** Ask better questions. Define inputs, contexts, constraints.  
  - **Backlog gate:** Requirements, risks, reviews.  
  - **Doing gate:** Unit/integration/system testing.  
  - **Done gate:** Verified artifacts, linkable proofs, public witnesses.
- **Red/blue collaboration:**  
  Opposed teams move the needle by challenging models and improving coherence.
- **Operations vs projects:**  
  - **Projects:** Time-boxed outcomes with milestone seeds.  
  - **Operations:** Ongoing stewardship with clear coverage metrics.
