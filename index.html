Design Principles

Built differently

OBIX is not a React wrapper. Every principle below is a constraint that shapes the architecture — not a feature that can be toggled off.

DOA
Data-Oriented Architecture

Components are plain data structures. State is explicit, inspectable, and serialisable. No hidden closures. No reactive proxies. Every instance is deterministic.

A11Y
Accessibility-First

WCAG 2.2 is a hard constraint enforced at the policy layer. Touch targets, ARIA labels, focus management, and reduced-motion support are not optional. They cannot be forgotten.

FUD
FUD Mitigation

Failure modes are formalised: Autocomplete Attribute Neglect, Form Validation Timing errors, CLS from unsized containers, focus trap failures. Identified at definition time, not runtime.

⏸
State Halting

Instances that stabilise are automatically halted — no unnecessary re-renders. A halted instance signals terminal condition. Explicit resume when needed. The obix-core test suite has 31 cases covering this precisely.

ND
Neurodivergent-First

Sensory considerations are built in — reduced motion, high contrast, predictable interaction patterns, clear focus indicators. Not afterthoughts. Architecture.

OBI
Constitutional Foundation

Named from the Igbo word Obi — Heart and Soul. Software as a human rights infrastructure. Grounded in the OBINexus constitutional computing framework.

Quickstart

Start in three steps

No CLI, no generator, no framework configuration. Just components.

obix-core — register and dispatch TypeScript
import { ObixRuntime } from '@obinexusltd/obix-core';

const runtime = new ObixRuntime();

// Register a component definition
runtime.register({
  name: 'Counter',
  state: { count: 0 },
  actions: {
    increment: (s) => ({ count: s.count + 1 }),
    decrement: (s) => ({ count: s.count - 1 }),
    reset: () => ({ count: 0 }),
  }
});

// Create an instance
const id = runtime.create('Counter');

// Dispatch actions
runtime.dispatch(id, 'increment');
runtime.dispatch(id, 'increment');

// Read state
const inst = runtime.getInstance(id);
console.log(inst.state.count); // 2

Install obix-core
npm install @obinexusltd/obix-core
2
Register your component
Define name, initial state, and actions. Actions are pure functions — state in, delta out. No side effects in the definition.
3
Create instances and dispatch
Each call to create() returns a unique ID. Dispatch actions against that ID. State is updated, lifecycle hooks fire, revision increments.
